## 3. DELIVERABLES

There are four core documents. They serve different purposes and must not duplicate each other.

### 3.1 Appendix A � Master Comparable Schedule (Excel or Word table)
**Purpose:** The single source of truth for all comparable transactions. Every verified comp goes here. Investment committee members click through the evidence links to verify claims.

**Structure:**
- Organised into tiers (e.g. Tier 1: Developer Direct Comp, Tier 2: Peer Village NB, Tier 3: Subject Village Prime Resale, Tier 4: Inferior Stock Floor, Tier 5: Peer Village Resale)
- Tier headers show psf range and role (e.g. "Subject village value ceiling")
- 10 columns total: Scheme | Unit | Date | Price (�) | SqFt | �/psf | Svc Chg | Type | Notes | Evidence
- Column J (Evidence) = clickable hyperlink. Display text = the property address / unit name (e.g. "Sperrings, Witney Lane", "5 Hicks Close"). Target = the source URL (RM UUID detail page preferred).
- No separate raw-URL column. The hyperlinked cell serves both purposes � display for quick identification, target for click-through.
- �/psf column uses a live formula: `=IF(AND(D{row}<>"",E{row}<>"",E{row}<>0),D{row}/E{row},"")`
- After any save, run `recalc.py` to recalculate cached formula values

**Notes column (Col I) discipline:**
- Keep to one line � "why this is a comparable" only.
- Must include (a) scheme/stock descriptor, (b) size-match to a specific subject plot, (c) tier role in one phrase.
- Example format: "Prime Leafield detached character. Size-matched to Plots 4 & 5; establishes village prime-resale ceiling."
- Do NOT clutter Notes with verification commentary, sqft-source notes, EPC caveats, garage flags, or �/psf arithmetic. That belongs in the project working notes or the Forensic Report, not the Appendix.

**Excel technical rules (CRITICAL � learned from painful experience):**
- **openpyxl row insertion breaks hyperlink mapping.** NEVER insert rows into an existing Excel file with hyperlinks. Always rebuild the entire workbook from a complete master data array. This is non-negotiable.
- After any rebuild, verify every row's Col J cell has a valid hyperlink target. Run a programmatic check, not a visual scan.
- Use openpyxl with `--break-system-packages` flag for pip install.

### 3.2 Forensic Comparable Evidence Report (Word .docx)
**Purpose:** The narrative analysis document. Draws conclusions from the Appendix A data. Goes to the investment committee alongside the Appendix. Contains small summary tables but does NOT reproduce the full comparable schedule.

**Structure (mirror Alex's approved format):**
1. Executive Summary � key findings table, 2-3 sentence conclusions
2. Comparable Scheme Analysis � per-scheme sections with: background paragraph, transaction summary table, premium vs local market, bold Key Takeaway line
3. Local Market Analysis � baseline + subject location premium
4. Conclusions & Recommendations � premium summary table, positioning statement

**Says "see Appendix A for full transaction evidence" � does not embed 60 rows of data.**

### 3.3 Local Market Deep Dive (Word .docx)
**Purpose:** The wider evidence base that sits underneath the Appendix. Contains transactions that DIDN'T make the Appendix cut � additional sales per area, streets that didn't qualify, secondary villages, market floor evidence.

**CRITICAL: This document must contain ZERO overlap with Appendix A.** If a transaction is in the Appendix, it does not appear in the Deep Dive. The Deep Dive is additive research, not a repackaging of the same data.

**Structure:**
- Title + section numbering (continues from Appendix A section numbers)
- Per area: bold title, italic status line, evidence table, one-line avg + source note
- Summary table at end
- 3-4 sentence conclusion. Only references numbers from THIS document � does NOT reference Appendix A figures. The Forensic Report is where the two get tied together.

### 3.4 Accommodation Schedule (Excel)
**Purpose:** The full unit breakdown with pricing from AL and any agents (KF, CJ, etc.). Shows plot number, beds, type, sqft, and price per valuer, with �/psf formulas and GDV totals.

**Post-build verification checklist (run programmatically):**
1. Every �/psf cell = Price � Sqft (no hardcoded values)
2. Total GDV = SUM of all individual prices
3. Blended �/psf = Total GDV � Total Sqft
4. AL total matches expected figure (��10k for rounding)
5. Agent totals match expected figures
6. Unit count matches scheme total
7. No blank cells in data range

## 4. ALEX'S STYLE BENCHMARK

**All documents must mirror Alex's approved format from the outset.** Do not default to formal report prose and then rewrite later. Get it right first time.

### Formatting
- **Font:** Calibri 11pt body
- **Headers:** Dark blue (#2E5090 or #1F4E79), bold
- **Status lines:** Italic, one line under each section heading
- **Table headers:** White text on dark blue fill, or dark blue text on light grey (#D9E2F3) fill
- **Alternating row shading:** Light grey on data rows (optional)
- **Borders:** Light grey, thin
- **Page:** Portrait A4

### Writing style
- **2-3 sentence paragraphs MAX.** Never write 5-8 sentence essays.
- **Bold standalone Key Takeaway lines:** `Key Takeaway: [Scheme] establishes that [key insight].`
- **Bold standalone Premium lines:** `Premium Achieved: +XX% above local [Location] market.`
- **Shorthand throughout:** c.�663 psf, +5.6%, NB, LR, det, EOT, SC, pa. Never spell out "approximately 1,611 square feet" � write "c.1,611 sqft".
- **Conversational, not academic:** "The units are now very tired with dated finishes" not "The residential units exhibit characteristics of age-related degradation."
- **Agent dialogue tone:** "Agree with KF pricing" / "Think KF have underpriced" / "We sit middle."
- **Property-specific notes in justification columns:** "Compact 3-bed EOT; good family size, strong liquidity at ~�870k" NOT "EOT premium supported by Oakfields comparable evidence."
- **No promotional language.** This is forensic evidence for a bank monitor, not marketing for buyers. No "zenith of the market," "trophy asset with collector's appeal," "architectural masterpiece." Neutral valuer evidence language.
- **No over-reliance on provenance/heritage as a premium driver.** If the scheme has heritage significance, reference it once in the exec summary and let the transaction data speak.

## 5. DATA RULES

### 5.1 PSF Formula
**Price � SqFt.** That's it. Never overcomplicate. Never estimate � always calculate. Show the maths if asked.

### 5.2 Sqft hierarchy (priority order)
1. **Rightmove floor plan** from specific property detail page (check each unit's page individually)
2. **Developer brochure / specification sheet** � published sqft per house type (confirm "Total Internal Living Area" = habitable GIA excluding garages)
3. **Agent brochure / listing** � Zoopla, OnTheMarket, or agent website
4. **Planning application GIA schedule** � approved docs on council portal
5. **Bricksandlogic / StreetScan** � EPC-derived postcode average (state source)
6. **Chimnie** � EPC fallback. Lower confidence. Frequently understates sqft for NB vs developer brochures. Treat as approximate.
7. **No sqft found** � flag as DATA GAP. Do not guess.

**Never use a generic EPC figure when a specific unit's Rightmove listing has a floor plan.**

### 5.3 Price types
- **Achieved prices only** unless explicitly labelled. Land Registry confirmed via Rightmove house-prices, Zoopla, or Bricksandlogic.
- **Asking prices** must be clearly labelled "ASKING" in the Notes column and excluded from averaged �/psf calculations.
- **Marketing prices** from developer websites are asking prices � label accordingly.
- If a developer has withdrawn asking prices (e.g. Osprey), note this.

### 5.4 Link standards
- **Specific Rightmove detail page UUID links ONLY:** `rightmove.co.uk/house-prices/details/[UUID]`
- **Never generic postcode links:** `rightmove.co.uk/house-prices/[postcode].html` ? these are NOT investment committee standard
- If you cannot find the specific detail page UUID, provide the generic link but flag it: "Detail page not located � verify manually."
- Rayn adds/verifies RM UUID links manually. Claude provides candidate links for review; does not edit the sheet unless instructed.
- Display text in hyperlink column: "RM Detail" or "Zoopla Detail" or "OTM Detail" � keep it short.

### 5.5 Agent data verification
**Agent schedules require independent verification.** Previous project confirmed at least one Carter Jonas plot-to-type mapping error (Plot 7 Oakfields � CJ assigned 5-bed Charlton at 2,584 sqft; Rightmove floor plan showed 4-bed Spencer at 2,990 sqft, which initially appeared to disprove CJ, but developer brochure later confirmed CJ was right and the RM floor plan included the garage). Lessons:
- Always cross-check agent sqft against developer "Total Internal Living Area" (habitable GIA excluding garages)
- RM floor plans may include garages � check whether the stated sqft says "excluding garage" or not
- If there's any doubt, flag both figures and let Alex decide
- Never rely on a single source for sqft

### 5.6 Calculated averages
- When the Appendix tier header states an average (e.g. "�612 psf"), verify it by calculating from the individual transactions. If the header says �612 but the calculation yields �604, use the calculated figure.
- Blended averages across a tier = total value of all transactions � total sqft of all transactions. NOT the average of individual �/psf figures (which weights small and large units equally).

## 6. VERIFICATION DISCIPLINE

**These rules exist because previous deliverables had errors that cascaded.**

### 6.1 No hallucinating data
If you cannot verify a transaction against a Rightmove detail page, Land Registry record, or other primary source, do not include it. Flag it as "Unverified" or "Data Gap."

### 6.2 Stale references cascade
When ANY primary figure changes (a price, a sqft, a psf), search the ENTIRE document for every other place that figure or any derivative appears. Update all downstream references � key takeaways, summary tables, tier headers, conclusion paragraphs, exec summary � in the same pass. Not just the primary data cell.

### 6.3 Verify the output file, not the working copy
After building or editing any document, verify the fix against the actual file in `/mnt/user-data/outputs/` � not the working copy in `/home/claude`. Open, read, and confirm before telling Rayn it's done.

### 6.4 Batch fixes, then review
Rayn prefers to accumulate corrections before implementing, not fix-as-you-go. When reading through a document, keep a running numbered list of issues. Implement all fixes in one clean pass at the end.

### 6.5 Rayn handles minor edits
For small text changes (a word, a number, a note), provide the exact value/text for Rayn to paste in himself. Don't burn usage on trivial edits unless instructed.

### 6.6 Floor plan images
When Rayn provides floor plan images, calculate sqft directly from visible dimensions without waiting to be asked. Flag if the stated sqft includes/excludes garage.

### 11.3 Key lessons that shaped the rules above

**CJ Plot 7 saga:** CJ said Plot 7 Oakfields was a 5-bed Charlton at 2,584 sqft (�619 psf). Rayn found the RM floor plan showing 2,990 sqft � we declared CJ wrong. Then Elivia's developer brochure confirmed 2,584 sqft was correct habitable GIA excluding garages � the RM floor plan had included the integral garage. CJ was right all along. **Lesson:** Always check whether sqft includes/excludes garages. Cross-reference multiple sources before declaring an error.

**openpyxl hyperlink disaster:** Inserted 3 rows into a 60-row Excel sheet. Every hyperlink in column J shifted to the wrong row. Had to rebuild the entire workbook from scratch. **Lesson:** Never insert rows. Always rebuild from a complete master data array.

**Oakfields blended average:** Tier header stated �612 psf. Actual calculation from 10 individual transactions = �604 psf. **Lesson:** Always verify header averages against individual row calculations.

**Chimnie sqft understatement:** Chimnie showed Castlefield 3-beds at ~1,140 sqft. Osprey developer brochure confirmed 922-992 sqft. **Lesson:** Chimnie frequently overstates or understates vs developer data. Use as fallback only.

**Forensic Report tone mismatch:** First draft used 5-8 sentence academic paragraphs with Lutyens/Jekyll provenance oversell. Alex's style is 2-3 sentence punchy notes with shorthand. Had to completely rebuild to match. **Lesson:** Read Alex's approved files BEFORE writing, not after.

**Deep Dive vs Appendix duplication:** First Deep Dive prompt produced sections 1-7 that were straight copies of Appendix A data. Completely pointless. **Lesson:** Deep Dive must have an explicit exclusion list of every Appendix A property.

**Gemini hallucinations:** Gemini claimed specific transaction prices and dates that didn't match Land Registry. Also missed qualifying sales (e.g. 5 Chequers Lane, �690k, Feb 2024 � clearly met the criteria but Gemini didn't find it). **Lesson:** Always verify Gemini output against primary sources before including in any document.

**Stale reference cascade:** Changed a headline psf figure but forgot to update the Key Takeaway, summary table, and conclusion paragraph that referenced the old number. Alex caught it. **Lesson:** After ANY data change, grep the entire document for every instance of the old value.

**Rightmove developer pages going dead:** Broadoaks Park and several other RM developer branch pages returned 404 after the schemes sold out. Had to find OTM/Zoopla fallback links. **Lesson:** OTM and Zoopla pages sometimes survive longer than RM developer pages.

**Paynes Park House:** Initially included as a heritage benchmark. It's a commercial property, not residential. Would have been torn apart by a Red Book valuer. **Lesson:** Always verify the use class before including any comp.
