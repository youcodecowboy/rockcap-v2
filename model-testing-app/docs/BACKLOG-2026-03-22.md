## Execution Order

### Tier 1 — Quick Wins (Agent: Main)

- [x] **UIX-01** | Fix white text on white background for user names and projects
  - Priority: High | Effort: Low | Category: UI/UX
  - **Commit:** `30faf8d`
  - **Root cause:** The `@` mention autocomplete dropdowns (`NoteMentionList.tsx` and `MentionAutocomplete.tsx`) had no explicit `color` set. The containers and buttons relied on inherited text color. Because tippy.js portals append to `document.body`, and Tailwind v4 CSS layers can cause unexpected color inheritance, the text inherited `color: white` from parent contexts (e.g., the Notes page header with `bg-blue-600 text-white`).
  - **Fix:** Added explicit `text-gray-900 dark:text-zinc-100` to both the dropdown container `<div>` and each `<button>` element in `NoteMentionList.tsx` (Notes TipTap editor) and `MentionAutocomplete.tsx` (Chat input). This ensures mention autocomplete text is always readable regardless of portal parent context.

- [x] **UIX-02** | Remove red trash can icon from line items
  - Priority: Low | Effort: Low | Category: UI/UX
  - **Commit:** `b43aaf6`
  - **Root cause:** Task line items in `ClientTasksTab.tsx` and `ProjectTasksTab.tsx` had a hover-revealed red `Trash2` icon button (`opacity-0 group-hover:opacity-100`) positioned absolutely in the top-right corner of each task row.
  - **Fix:** Removed the entire `<button>` with the red Trash2 icon from the `TaskListItem` component in both files. The `group` class on the parent div was also removed (no longer needed). Delete functionality remains available through the task detail panel when a task is selected. Cleaned up the unused `onDelete` destructured prop with a `_` prefix.

- [x] **LIB-03** | Fix 'View All' in recent documents to navigate to document library
  - Priority: Medium | Effort: Low | Category: Doc Library
  - **Root cause:** The "View All" buttons in `ProjectOverviewTab.tsx` and `ClientOverviewTab.tsx` used `router.push()` to navigate to `?tab=documents`. However, since the page component was already mounted, `useState(initialTab)` only read the URL param once on mount. Subsequent `router.push` URL changes didn't update the `activeTab` React state, so the tab never switched.
  - **Fix:** Added an `onTabChange` callback prop to both `ProjectOverviewTab` and `ClientOverviewTab`. The parent pages (`[projectId]/page.tsx` and `[clientId]/page.tsx`) pass their existing `handleTabChange` function which updates both the React state AND the URL. All "View All" buttons now call `onTabChange?.('documents')` / `onTabChange?.('checklist')` / `onTabChange?.('tasks')` / `onTabChange?.('projects')` instead of `router.push`.

- [ ] **FIL-01** | Fix download filename preservation
  - Priority: High | Effort: Low | Category: File Mgmt
  - When downloading a file, preserve the application-assigned name
  - Currently generates a random string of numbers instead of the display name

## Architecture Notes for Agents

- **Stack:** Next.js 16 + Convex backend + Anthropic Claude API
- **Classification pipeline:** V4 at `src/v4/`, API route at `src/app/api/v4-analyze/`
- **Shared reference library:** `src/lib/references/` — tag-based resolver for document classification
- **Filing flow:** `src/lib/bulkQueueProcessor.ts` → `/api/v4-analyze`
- **Chat system:** Claude Haiku 4.5 via `@anthropic-ai/sdk`, tools in `src/lib/tools/`
- **Build check:** Always run `npx next build` before committing
- **Commit rule:** Push to GitHub after each change for code review ease
